## 二、致命问题（必须修复后才能上线）

### 🔴 1. 环境变量未配置
后端 API 依赖 4 个环境变量，Vercel 必须配置：
```
SILICONFLOW_API_KEY   ← SiliconFlow API Key（AI 生成）
GMAIL_USER            ← Gmail 发件邮箱
GMAIL_PASS            ← Gmail 应用密码（不是登录密码）
JWT_SECRET            ← JWT 密钥（随机字符串，建议32位以上）
```

**问题**：没有这些变量，`/api/send-code`、`/api/ai` 等所有 API 都会失败。

**修复方法**：Vercel Dashboard → Project → Settings → Environment Variables，逐一配置。

---

### 🔴 2. Gmail 发信配置问题
`send-code.js` 使用 nodemailer 的 Gmail 认证：
```javascript
const transporter = nodemailer.createTransport({
  service: 'gmail',
  auth: { user: process.env.GMAIL_USER, pass: process.env.GMAIL_PASS }
});
```
**问题**：Gmail 不再支持"应用密码"以外的 SMTP 发送。如果 `GMAIL_PASS` 不是 Gmail 应用密码，发件会失败。

**验证步骤**：在 Vercel 环境变量配置好后，需要测试发件是否正常。

---

### 🔴 3. 额度控制有漏洞
`use-quota.js` 的逻辑：
```javascript
// 前端调用 use-quota，然后调用 ai
await fetch('/api/use-quota', ...);
const res = await fetch('/api/ai', ...);  // ai.js 没有再次检查额度！
```

**漏洞**：攻击者可以直接调用 `/api/ai` 绕过额度控制。`ai.js` 根本不检查 token，任何人都可以无限使用。

**修复**：在 `ai.js` 内部加入额度验证：
```javascript
// ai.js 顶部加入
const token = req.headers.authorization?.replace('Bearer ', '');
if (token) {
  // 验证 token 和额度，额度用完返回 429
}
```

## 三、中等问题（影响体验）

### 🟡 4. 首页定价链接是死链
index.html nav 有 `#pricing` 锚点，但页面中只有 `#tools` 和 footer，没有定价页。点击"定价 Pricing"无效。

**修复**：要么做定价页，要么把链接改成锚到 `#tools`。

---

### 🟡 5. 隐私政策/条款样式与主站不统一
- index.html 和工具页：白色浅色主题（`#fafbfc` 背景）
- privacy.html / terms.html：**暗色主题**（`#0a0a0f` 背景 + 噪点纹理 + 深绿accent）

点击 footer 的"隐私政策"链接，页面风格突变，用户体验割裂。

**修复**：统一成浅色主题，或反过来把主页也改成暗色。

---

### 🟡 6. 工具页"翻译"名不副实
`tools/translate/index.html` 从文件名看是"翻译工具"，但实际上这个页面应该是一个翻译界面，需要检查内容。如果工具未完成，应该标注"敬请期待"或先做出来。

---

### 🟡 7. 邮箱验证弹窗没有关闭按钮
每个工具页的 modal（验证弹窗）只能点遮罩关闭，没有右上角 × 按钮。部分用户可能不知道可以点遮罩关闭。

**修复**：加一个关闭按钮（modal.js 里已有 `.modal-close` 样式但没有对应按钮）。

---

### 🟡 8. 没有使用统计和用户引导
新用户第一次来，没有引导——不知道工具怎么用，不知道有5次免费额度，不知道需要验证邮箱。

**建议**：首次访问弹出简短引导 tooltip（3秒消失），或首页底部加一行说明文字。
